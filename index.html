2. About this document

This document maps every fan-facing screen in even-web (legacy fan app at app.even.biz / www.even.biz) and proposes E2E test flows for the E2E Validation Gate initiative.

Built from a local stack investigation: even-web (localhost:3000) connected to even-back Docker (localhost:5000) with staging DB (1,120 users, 681 releases). All findings are proven, not assumed.

Purpose: Align team on screen naming, E2E test scope, and priority before writing test code. Same approach used for Backstage Screen Map.

7. Phased rollout

Phase 1: Foundation (5 specs) — Blocks EWM-M1

  • Smoke: app loads against local backend
  • Auth bypass: E2E bypass creates session (proven working)
  • Session persistence: survives navigation + refresh
  • Release page: /r/[slug] renders (after mediaURL fix)
  • Explore page: /explore loads with releases

Phase 2: Core Flows (+ 3 specs) — Blocks EWM-M2/M3

  • Artist profile: /artists/[slug] renders with releases
  • Auth redirect: unauthenticated → login → return to page
  • Library: /users shows owned releases

Phase 3: Revenue (+ 3 specs) — Blocks EWM-M6

  • Checkout visual: /checkout/[slug] renders with tiers
  • Post-purchase portal: access content visible
  • Audio player: player component renders with track list
